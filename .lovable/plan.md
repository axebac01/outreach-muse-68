

## Step 3: Database + Logic

### Summary
Set up Lovable Cloud (Supabase) with five tables, RLS policies, and refactor the app from mock data to real database-backed CRUD with usage limits.

### Prerequisites
- **Enable Lovable Cloud** on this project (needed before creating migrations)

### Database Schema

```text
profiles
├── id (uuid, PK, FK → auth.users)
├── full_name (text)
├── plan (text, default 'starter')  -- 'starter' | 'growth'
├── created_at (timestamptz)

campaigns
├── id (uuid, PK)
├── user_id (uuid, FK → auth.users, NOT NULL)
├── name (text)
├── target_audience (text)
├── product (text)
├── offer (text)
├── tone (text)
├── status (text, default 'draft')  -- 'draft' | 'generated'
├── created_at (timestamptz)

leads
├── id (uuid, PK)
├── campaign_id (uuid, FK → campaigns ON DELETE CASCADE)
├── user_id (uuid, FK → auth.users, NOT NULL)
├── full_name (text)
├── company (text)
├── role (text)
├── website (text)
├── linkedin_url (text)
├── notes (text)
├── created_at (timestamptz)

generated_outreach
├── id (uuid, PK)
├── lead_id (uuid, FK → leads ON DELETE CASCADE)
├── user_id (uuid, FK → auth.users, NOT NULL)
├── subject_line (text)
├── opener (text)
├── cold_email (text)
├── follow_up_1 (text)
├── follow_up_2 (text)
├── status (text, default 'pending')  -- 'pending' | 'approved'
├── created_at (timestamptz)

usage_tracking
├── id (uuid, PK)
├── user_id (uuid, FK → auth.users, NOT NULL)
├── action (text)  -- 'campaign_created' | 'outreach_generated'
├── created_at (timestamptz)
```

### RLS Policies
All tables: users can only SELECT/INSERT/UPDATE/DELETE their own rows (`user_id = auth.uid()`). Profiles use `id = auth.uid()`.

### Plan Limits (enforced in app logic)
| Resource | Starter (Free) | Growth (€99/mo) |
|----------|----------------|-----------------|
| Campaigns | 1 | Unlimited |
| Leads per campaign | 10 | Unlimited |
| Outreach generations | 10/month | Unlimited |

### Auto-Profile Creation
Database trigger on `auth.users` INSERT → creates a `profiles` row with `plan = 'starter'`.

### App Code Changes

1. **Supabase client setup** — `src/integrations/supabase/client.ts` (auto-generated by Lovable Cloud)

2. **Auth context** — Create `AuthProvider` with `onAuthStateChange`, protect dashboard/campaign/outreach routes

3. **Custom hooks** (replace mock data):
   - `useCampaigns()` — fetch user's campaigns
   - `useCampaign(id)` — single campaign with leads
   - `useLeads(campaignId)` — leads for a campaign
   - `useGeneratedOutreach(leadId)` — outreach for a lead
   - `useUsage()` — check limits
   - Mutation hooks: `useCreateCampaign`, `useCreateLead`, `useGenerateOutreach`, `useRegenerateOutreach`, `useApproveOutreach`

4. **Page refactors**:
   - `Dashboard.tsx` — use `useCampaigns()` instead of `sampleCampaigns`
   - `CreateCampaign.tsx` — use `useCreateCampaign` mutation, check campaign limit
   - `CampaignDetails.tsx` — use `useCampaign`, `useCreateLead`, check lead limit
   - `Outreach.tsx` — use `useGeneratedOutreach`, add approve button
   - `Login.tsx` / `Signup.tsx` — wire to Supabase auth

5. **Generate outreach** — For now, generates placeholder emails server-side via an edge function (AI integration deferred). Records usage.

6. **Upgrade prompts** — Show upgrade banner when limits reached, linking to `/pricing`
